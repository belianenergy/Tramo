# EnergyOS — Landing MVP para gestores de apartamentos turísticos (Stitch v7)

Crea una landing page DESKTOP de alta fidelidad para **EnergyOS**, un servicio B2B para gestoras con **10+ apartamentos turísticos** en España.

## Objetivo de negocio
Convertir a gestoras de apartamentos turísticos que quieren automatizar confort y reducir consumo sin complicarse con tecnología.

## Idioma
Todo el copy en **castellano**. Tono profesional, concreto y orientado a ahorro operativo. Nada de jerga de IA como protagonista.

## Posicionamiento principal
**Automatización de confort + control de consumo para alquiler turístico**:
- gestión energética antes del check-in,
- apagado o modo bajo consumo tras check-out,
- alertas de consumo anómalo,
- informe mensual claro para propietarios.

## Posicionamiento secundario
Optimización de tarifa eléctrica. Debe aparecer como beneficio secundario, no como hero.

## Lead magnet premium
Batería / arbitraje energético como bloque premium de futuro o upsell, pero **no debe dominar el hero**.

## Estilo visual
- Modo claro.
- Profesional, premium, tipo Linear/Stripe.
- Fondo blanco roto #FAFAFA, tarjetas #FFFFFF, bordes #E5E7EB.
- Verde esmeralda #10B981 como acento principal; ámbar #F59E0B para alertas; rojo #EF4444 para anomalías.
- Tipografía Inter; números con estilo tabular/monospace.
- Mucho aire, grid preciso, tarjetas limpias, sombras muy suaves.
- Nada genérico de inmobiliarias, comunidades de vecinos ni IA como foco central.
- Debe parecer una herramienta operativa seria para property managers.

## Estructura obligatoria

### 1) Top nav
Logo "EnergyOS" con icono simple de energía. Links: "Cómo funciona", "Automatizaciones", "Informes", "Demo". CTA: "Solicitar demo".

### 2) Hero
Headline fuerte:
"Reduce y controla la energía de tus alojamientos turísticos"

Subheadline:
"Centraliza consumo, reservas, tarifas e informes para ahorrar, detectar anomalías y valorar baterías o arbitraje energético."

CTAs:
- Primario: "Solicitar demo"
- Secundario: "Ver ejemplo de informe"

Visual hero a la derecha: mockup de producto con:
- lista de 5 apartamentos,
- estados: "Pre-check-in", "Ocupado", "Post-check-out", "Alerta consumo",
- una alerta roja: "Consumo alto 3h después del check-out",
- métrica grande: "-18% consumo fuera de estancia",
- mini timeline check-in/check-out.

### 3) Banda de confianza / ICP
Pequeña franja: "Diseñado para gestoras con 10+ apartamentos turísticos" + badges: "Airbnb", "Booking", "PMS", "Telegram/WhatsApp alerts".

### 4) Tres pilares principales
Tarjetas con icono y micro-mockup:
1. "Automatización por reservas" — activa clima a tiempo para que el huésped llegue a una vivienda preparada.
2. "Optimización de tarifas" — apaga o reduce gestión energética automáticamente cuando termina la estancia.
3. "Baterías y arbitraje" — detecta A/C, termo o calefacción funcionando cuando no debería.

### 5) Sección "Cómo funciona"
Tres pasos horizontales:
1. "Conectamos calendario/PMS" — reservas, check-ins y check-outs.
2. "Medimos y automatizamos" — sensores/medidores y reglas por vivienda.
3. "Informamos al equipo" — alertas y resumen mensual para propietarios.

### 6) Dashboard / informe mensual
Sección con mockup grande de informe para propietarios:
- ahorro mensual estimado,
- consumo fuera de estancia,
- top 3 anomalías,
- recomendaciones de tarifa,
- estado por apartamento.
Debe reforzar que sirve para justificar gestión profesional ante propietarios.

### 7) Optimización de tarifa secundaria
Bloque más pequeño: "Además, revisamos si tu tarifa encaja con el uso real".
Mostrar comparativa simple: tarifa actual vs recomendada, ahorro estimado.

### 8) Premium / batería-arbitraje
Bloque premium discreto al final: "Para carteras grandes: simulación de batería y arbitraje".
Debe sentirse como upsell avanzado, no como producto principal.

### 9) Formulario lead
Card final con título:
"¿Gestionas 10+ apartamentos? Te mostramos dónde se escapa el consumo."
Campos:
- Nombre
- Email
- Teléfono
- Nº de apartamentos
- Ciudad
- PMS usado (opcional)
CTA: "Solicitar demo de 20 minutos"
Texto de confianza: "Sin compromiso. No tocamos instalaciones ni automatizaciones sin autorización."

### 10) Footer
Simple: EnergyOS, "Automatización energética para alquiler turístico", links básicos.

## Restricciones
- No mencionar comunidades de vecinos/fincas como caso principal.
- No hacer que la IA sea el centro de la propuesta.
- No usar estética oscura ni cyberpunk.
- No usar stock inmobiliario genérico como protagonista.
- El hero debe vender confort + consumo, no arbitraje.

Entrega una pantalla completa, visualmente terminada, con componentes realistas, copy final, datos de ejemplo y jerarquía clara.
